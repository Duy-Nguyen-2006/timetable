# code_executor — Python Execution Host for AI Agent

This binary is used **only** during the AI agent loop (Coder self-debug phase).

## Responsibilities
- Receive complete Python source code via **stdin**.
- Execute it in the current working directory (where `input.json` has already been written by the desktop app).
- Enforce hard timeout (180 seconds recommended).
- **Always** output exactly **one JSON object** on stdout matching the `ExecutionResult` interface (see `src/features/timetable/ai/types.ts`).

## Expected Behavior from Generated Code

The LLM (Coder) must produce a complete, runnable Python script that:

1. Reads the problem definition from `./input.json` (UTF-8 JSON).
2. Uses `ortools.sat.python.cp_model` to solve the timetable.
3. On success:
   - Prints the string `SOLUTION FOUND` somewhere in stdout (case-insensitive detection).
   - Outputs (either via print or to a file we can parse) a JSON object with at least:
     ```json
     {
       "classes": ["6A", "6B", ...],
       "days": ["Thứ 2", ...],
       "periods": [1,2,3,...],
       "schedule": [
         { "class": "6A", "day": "Thứ 2", "period": 1, "subject": "Toán", "teacher": "Sơn" }
       ]
     }
     ```
4. On any failure, normal Python exceptions are acceptable (they will be captured in `stderr`).

## Input JSON Shape (input.json)

See `src/features/timetable/ai/types.ts` → `AgentInputPayload`.

Minimal expected fields (the generated code should be defensive):
- `days`, `sessions`, `periodCounts`
- `assignments[]`
- `constraints[]`
- `deletedPeriods`

## Building the Binary

```bash
pyinstaller python/code_executor.py \
  --onefile \
  --name code_executor \
  --distpath python-dist/code_executor \
  --workpath build/pyinstaller \
  --specpath build/pyinstaller
```

The resulting `code_executor` (or `.exe` on Windows) will be bundled via `electron-builder` `extraResources`.

## Security Note

This executor runs untrusted code generated by an LLM. It should only be used inside the desktop app where the user has explicitly provided their own LLM credentials. No network access is given to the executed code by default.
